## 📌 스터디 소개

알고리즘 실력 향상을 목표로 함께 문제를 풀고, 서로의 풀이를 리뷰하는 스터디입니다.  
백준, 프로그래머스, LeetCode 등 플랫폼 제한 없이 자유롭게 참여할 수 있습니다.

---

## 🗂 폴더 구조

각 스터디원은 **본인 이름의 폴더** 안에서만 작업합니다.

```
algorithm/
├── README.md
├── .gitignore
├── .github/
│   └── PULL_REQUEST_TEMPLATE.md
│
├── 김가원/
│   ├── BOJ_1234_문제이름.py
│   └── PGS_카카오_문제이름.java
│
├── 홍길동/
│   ├── BOJ_2468_안전영역.cpp
│   └── LeetCode_42_TrappingRainWater.py
│
└── 이몽룡/
    └── BOJ_1753_최단경로.java
```

> **파일 네이밍 예시**
> - 백준: `BOJ_{번호}_{문제명}.{확장자}`
> - 프로그래머스: `PGS_{문제명}.{확장자}`
> - LeetCode: `LeetCode_{번호}_{문제명}.{확장자}`

---

## ⚙️ 운영 방식

```
각자 브랜치에서 작업 후 Main으로 PR 날리기
```

1. `main` 브랜치에서 **본인 이름으로 브랜치** 생성
   ```bash
   git checkout -b 김가원
   ```
2. 본인 폴더 안에 풀이 파일 추가
3. 커밋 & 푸시 후 **`main`으로 PR** 생성
4. 최소 1명 이상의 **Approve** 후 머지
